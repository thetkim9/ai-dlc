## 1. 기능 요구사항 (Functional Requirements)

### 1.1 고객용 기능

#### FR-C-01: 테이블 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호 입력)
- 설정 정보를 localStorage에 저장
- 이후 앱 접속 시 저장된 정보로 자동 로그인
- 로그인 성공 시 테이블 세션 토큰 발급

#### FR-C-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면으로 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동 (탭 또는 사이드바)
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- localStorage에 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C-04: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시: 주문 번호 표시 (5초) → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장/테이블 식별 정보, 메뉴 목록(메뉴명/수량/단가), 총 금액, 세션 ID

#### FR-C-05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태
- 주문 상태 실시간 업데이트 (SSE)
- 페이지네이션 또는 무한 스크롤

### 1.2 관리자용 기능

#### FR-A-01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증
- 16시간 세션 유지 (브라우저 새로고침 시 유지)
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장
- 로그인 시도 횟수 제한

#### FR-A-02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃 (테이블별 카드)
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A-03: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호 설정, 16시간 세션 생성)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 테이블 이용 완료 처리:
  - 확인 팝업
  - 현재 세션 주문 → 과거 이력으로 이동
  - 테이블 주문 목록 및 총 주문액 0으로 리셋
- 과거 주문 내역 조회 (날짜 필터, 시간 역순)

#### FR-A-04: 메뉴 관리 (CRUD)
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 파일 업로드
- 메뉴 수정
- 메뉴 삭제
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2.1 성능
- NFR-P-01: 실시간 주문 업데이트 2초 이내 반영
- NFR-P-02: 메뉴 조회 API 응답 1초 이내
- NFR-P-03: 소규모 기준 (테이블 10개, 메뉴 30개) 동시 접속 처리

### 2.2 보안
- NFR-S-01: JWT 토큰 기반 관리자 인증
- NFR-S-02: 비밀번호 bcrypt 해싱
- NFR-S-03: 로그인 시도 횟수 제한 (brute force 방지)
- NFR-S-04: 테이블 세션 토큰으로 고객 요청 인증
- NFR-S-05: 민감 정보 환경 변수 관리 (DB 비밀번호, JWT secret 등)

### 2.3 사용성
- NFR-U-01: 터치 친화적 UI (최소 44x44px 버튼)
- NFR-U-02: 직관적인 카드 형태 레이아웃
- NFR-U-03: 명확한 에러 메시지 및 피드백

### 2.4 운영
- NFR-O-01: Docker Compose로 전체 스택 실행 가능
- NFR-O-02: 환경 변수 기반 설정 관리 (.env 파일)
- NFR-O-03: 이미지 파일 Docker volume으로 영속성 보장

## 3. 데이터 모델 개요

### 주요 Entity
- **Store**: 매장 정보 (식별자, 이름, 관리자 계정)
- **Table**: 테이블 정보 (번호, 비밀번호, 현재 세션 ID)
- **TableSession**: 테이블 세션 (시작/종료 시각, 상태)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 정보 (이름, 가격, 설명, 이미지, 카테고리, 순서)
- **Order**: 주문 (세션 ID, 테이블, 상태, 총 금액, 시각)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
