# User Stories - 테이블오더 서비스

## 분류 방식: Feature-Based (Epic → Story 계층)
## Acceptance Criteria: 상세 수준 (Given/When/Then + 엣지 케이스 + 에러 시나리오)
## 우선순위: MoSCoW (Must Have / Should Have / Could Have)

---

# 고객용 (Customer) Epics

---

## Epic C1: 테이블 접속 및 인증

> 고객이 별도 로그인 없이 테이블 태블릿에서 즉시 주문 화면에 접근할 수 있다.

### Story C1-1: 테이블 자동 로그인
**As a** 고객으로서,
**I want** 태블릿을 켜면 자동으로 주문 화면이 표시되기를,
**So that** 별도 로그인 절차 없이 즉시 메뉴를 볼 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 시나리오*
- Given 관리자가 테이블 초기 설정을 완료하여 localStorage에 인증 정보가 저장되어 있을 때
- When 고객이 브라우저를 열거나 페이지를 새로고침하면
- Then 로그인 화면 없이 바로 메뉴 화면이 표시된다
- And 화면 상단에 테이블 번호가 표시된다

*세션 만료 시나리오*
- Given 테이블 세션이 만료되었을 때
- When 고객이 페이지에 접근하면
- Then "현재 이용 가능한 세션이 없습니다" 안내 메시지가 표시된다
- And 관리자에게 문의하도록 안내한다

*초기 설정 미완료 시나리오*
- Given localStorage에 인증 정보가 없을 때
- When 고객이 페이지에 접근하면
- Then 초기 설정 화면(매장 식별자, 테이블 번호, 비밀번호 입력)이 표시된다

*네트워크 오류 시나리오*
- Given 저장된 인증 정보가 있지만 서버 연결이 불가할 때
- When 자동 로그인을 시도하면
- Then "서버에 연결할 수 없습니다. 잠시 후 다시 시도해 주세요." 메시지가 표시된다
- And 재시도 버튼이 제공된다

## Epic C2: 메뉴 탐색

> 고객이 매장의 메뉴를 카테고리별로 쉽게 탐색하고 상세 정보를 확인할 수 있다.

### Story C2-1: 카테고리별 메뉴 조회
**As a** 고객으로서,
**I want** 카테고리별로 정리된 메뉴 목록을 보기를,
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 시나리오*
- Given 로그인된 고객이 메뉴 화면에 있을 때
- When 화면이 로드되면
- Then 카테고리 탭/목록이 상단에 표시된다
- And 첫 번째 카테고리의 메뉴들이 카드 형태로 표시된다
- And 각 메뉴 카드에는 이미지, 메뉴명, 가격이 표시된다

*카테고리 전환 시나리오*
- Given 고객이 메뉴 화면에 있을 때
- When 다른 카테고리 탭을 터치하면
- Then 해당 카테고리의 메뉴 목록으로 즉시 전환된다
- And 선택된 카테고리가 시각적으로 강조된다

*메뉴 없는 카테고리 시나리오*
- Given 특정 카테고리에 메뉴가 없을 때
- When 해당 카테고리를 선택하면
- Then "해당 카테고리에 메뉴가 없습니다" 메시지가 표시된다

*이미지 로드 실패 시나리오*
- Given 메뉴 이미지 URL이 유효하지 않을 때
- When 메뉴 카드가 표시될 때
- Then 기본 placeholder 이미지가 표시된다
- And 메뉴명과 가격은 정상 표시된다

### Story C2-2: 메뉴 상세 정보 확인
**As a** 고객으로서,
**I want** 메뉴 카드를 터치하여 상세 정보를 보기를,
**So that** 메뉴 설명을 읽고 주문 여부를 결정할 수 있다.

**Priority**: Should Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 시나리오*
- Given 고객이 메뉴 목록을 보고 있을 때
- When 메뉴 카드를 터치하면
- Then 메뉴 상세 모달/화면이 표시된다
- And 메뉴명, 가격, 설명, 이미지(크게)가 표시된다
- And 수량 선택 및 장바구니 담기 버튼이 제공된다

*장바구니 담기 시나리오*
- Given 고객이 메뉴 상세 화면에서 수량을 선택했을 때
- When "장바구니 담기" 버튼을 터치하면
- Then 해당 메뉴가 장바구니에 추가된다
- And 성공 피드백(토스트 메시지 또는 애니메이션)이 표시된다
- And 상세 화면이 닫히고 메뉴 목록으로 돌아간다

## Epic C3: 장바구니 관리

> 고객이 주문 전 선택한 메뉴를 담고 수정할 수 있다.

### Story C3-1: 장바구니에 메뉴 추가 및 수량 조절
**As a** 고객으로서,
**I want** 메뉴를 장바구니에 담고 수량을 조절하기를,
**So that** 원하는 메뉴와 수량을 확정하고 주문할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*메뉴 추가 시나리오*
- Given 고객이 메뉴 화면에 있을 때
- When 메뉴 카드의 "+" 버튼을 터치하면
- Then 해당 메뉴가 장바구니에 1개 추가된다
- And 장바구니 아이콘에 총 수량이 업데이트된다
- And 이미 담긴 메뉴를 다시 추가하면 수량이 증가한다

*수량 조절 시나리오*
- Given 장바구니에 메뉴가 담겨 있을 때
- When "+" 버튼을 터치하면 수량이 1 증가한다
- When "-" 버튼을 터치하면 수량이 1 감소한다
- When 수량이 1인 상태에서 "-"를 터치하면 해당 메뉴가 장바구니에서 삭제된다

*총 금액 계산 시나리오*
- Given 장바구니에 여러 메뉴가 담겨 있을 때
- When 수량이 변경될 때마다
- Then 총 금액이 실시간으로 재계산되어 표시된다

*페이지 새로고침 유지 시나리오*
- Given 장바구니에 메뉴가 담겨 있을 때
- When 페이지를 새로고침하면
- Then 장바구니 내용이 그대로 유지된다 (localStorage 저장)

### Story C3-2: 장바구니 비우기
**As a** 고객으로서,
**I want** 장바구니를 한 번에 비우기를,
**So that** 처음부터 다시 메뉴를 선택할 수 있다.

**Priority**: Should Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 시나리오*
- Given 장바구니에 메뉴가 담겨 있을 때
- When "장바구니 비우기" 버튼을 터치하면
- Then 확인 팝업이 표시된다
- When 확인을 누르면
- Then 장바구니가 비워지고 총 금액이 0원으로 표시된다

*빈 장바구니 시나리오*
- Given 장바구니가 비어 있을 때
- Then "장바구니 비우기" 버튼이 비활성화되거나 표시되지 않는다

## Epic C4: 주문 생성

> 고객이 장바구니의 메뉴를 실제 주문으로 확정할 수 있다.

### Story C4-1: 주문 확정
**As a** 고객으로서,
**I want** 장바구니 내용을 확인하고 주문을 확정하기를,
**So that** 주방에 주문이 전달되고 음식을 받을 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 주문 시나리오*
- Given 장바구니에 1개 이상의 메뉴가 담겨 있을 때
- When "주문하기" 버튼을 터치하면
- Then 주문 내역 확인 화면이 표시된다 (메뉴 목록, 수량, 단가, 총 금액)
- When "주문 확정" 버튼을 터치하면
- Then 서버에 주문이 전송된다
- And 주문 번호가 표시된 성공 화면이 5초간 표시된다
- And 장바구니가 자동으로 비워진다
- And 5초 후 메뉴 화면으로 자동 리다이렉트된다

*빈 장바구니 주문 시도 시나리오*
- Given 장바구니가 비어 있을 때
- Then "주문하기" 버튼이 비활성화된다

*주문 실패 시나리오*
- Given 주문 확정 버튼을 눌렀을 때 서버 오류가 발생하면
- Then "주문에 실패했습니다. 다시 시도해 주세요." 에러 메시지가 표시된다
- And 장바구니 내용은 그대로 유지된다
- And 재시도 버튼이 제공된다

*네트워크 오류 시나리오*
- Given 주문 전송 중 네트워크가 끊기면
- Then 로딩 인디케이터가 표시되다가 타임아웃 후 에러 메시지가 표시된다
- And 장바구니 내용은 유지된다

## Epic C5: 주문 내역 조회

> 고객이 현재 테이블 세션의 주문 이력과 상태를 확인할 수 있다.

### Story C5-1: 현재 세션 주문 내역 조회
**As a** 고객으로서,
**I want** 내가 주문한 내역을 확인하기를,
**So that** 어떤 메뉴를 주문했는지, 총 얼마인지 파악할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*정상 시나리오*
- Given 고객이 주문 내역 탭을 선택했을 때
- Then 현재 테이블 세션의 주문 목록이 시간 순으로 표시된다
- And 각 주문에는 주문 번호, 주문 시각, 메뉴 목록, 수량, 금액, 상태가 표시된다

*이전 세션 주문 제외 시나리오*
- Given 테이블이 이용 완료 처리 후 새 세션이 시작되었을 때
- When 새 고객이 주문 내역을 조회하면
- Then 이전 세션의 주문은 표시되지 않는다

*주문 없는 시나리오*
- Given 현재 세션에 주문이 없을 때
- Then "아직 주문 내역이 없습니다" 메시지가 표시된다

### Story C5-2: 주문 상태 실시간 확인
**As a** 고객으로서,
**I want** 주문 상태가 실시간으로 업데이트되기를,
**So that** 음식이 언제 나오는지 파악할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민준)

**Acceptance Criteria**:

*실시간 업데이트 시나리오*
- Given 고객이 주문 내역 화면을 보고 있을 때
- When 관리자가 주문 상태를 변경하면 (대기중 → 준비중 → 완료)
- Then 페이지 새로고침 없이 상태가 자동으로 업데이트된다 (SSE)
- And 상태별로 다른 색상/아이콘으로 표시된다 (대기중: 회색, 준비중: 주황, 완료: 초록)

*SSE 연결 끊김 시나리오*
- Given SSE 연결이 끊겼을 때
- Then 자동으로 재연결을 시도한다
- And 재연결 중임을 사용자에게 표시한다

# 관리자용 (Admin) Epics

---

## Epic A1: 매장 인증

> 관리자가 자신의 매장 관리 시스템에 안전하게 접근할 수 있다.

### Story A1-1: 관리자 로그인
**As a** 매장 관리자로서,
**I want** 매장 식별자와 계정 정보로 로그인하기를,
**So that** 내 매장의 주문 및 메뉴를 관리할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 로그인 시나리오*
- Given 관리자가 로그인 화면에 있을 때
- When 유효한 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인 버튼을 클릭하면
- Then JWT 토큰이 발급되고 관리자 대시보드로 이동한다
- And 브라우저를 새로고침해도 로그인 상태가 유지된다 (16시간)

*잘못된 자격증명 시나리오*
- Given 관리자가 잘못된 비밀번호를 입력했을 때
- When 로그인 버튼을 클릭하면
- Then "매장 식별자, 사용자명 또는 비밀번호가 올바르지 않습니다" 메시지가 표시된다
- And 비밀번호 필드가 초기화된다

*로그인 시도 제한 시나리오*
- Given 5회 연속 로그인 실패 시
- Then "너무 많은 로그인 시도가 있었습니다. 잠시 후 다시 시도해 주세요." 메시지가 표시된다
- And 일정 시간(예: 15분) 동안 로그인이 차단된다

*세션 만료 시나리오*
- Given 관리자가 로그인한 지 16시간이 지났을 때
- When 관리자 화면에 접근하면
- Then 자동으로 로그아웃되고 로그인 화면으로 이동한다
- And "세션이 만료되었습니다. 다시 로그인해 주세요." 메시지가 표시된다

## Epic A2: 실시간 주문 모니터링

> 관리자가 모든 테이블의 주문 현황을 실시간으로 파악하고 관리할 수 있다.

### Story A2-1: 테이블별 주문 대시보드 조회
**As a** 매장 관리자로서,
**I want** 모든 테이블의 주문 현황을 한 화면에서 보기를,
**So that** 어느 테이블에서 주문이 들어왔는지 즉시 파악할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 시나리오*
- Given 관리자가 대시보드에 접속했을 때
- Then 테이블별 카드가 그리드 형태로 표시된다
- And 각 카드에는 테이블 번호, 현재 총 주문액, 최신 주문 미리보기가 표시된다
- And 주문이 없는 테이블은 빈 상태로 표시된다

*신규 주문 알림 시나리오*
- Given 관리자가 대시보드를 보고 있을 때
- When 새 주문이 들어오면 (SSE)
- Then 2초 이내에 해당 테이블 카드가 업데이트된다
- And 신규 주문 카드가 시각적으로 강조된다 (색상 변경 + 애니메이션)

*주문 상세 조회 시나리오*
- Given 관리자가 테이블 카드를 클릭했을 때
- Then 해당 테이블의 전체 주문 목록 상세 화면이 표시된다
- And 각 주문의 메뉴 목록, 수량, 금액, 상태가 표시된다

### Story A2-2: 주문 상태 변경
**As a** 매장 관리자로서,
**I want** 주문 상태를 변경하기를,
**So that** 고객이 주문 진행 상황을 알 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*상태 변경 시나리오*
- Given 관리자가 주문 상세 화면에 있을 때
- When 주문의 상태 버튼을 클릭하면 (대기중 → 준비중 → 완료)
- Then 상태가 즉시 업데이트된다
- And 고객 화면에도 실시간으로 반영된다 (SSE)

*상태 역방향 변경 시나리오*
- Given 주문 상태가 "완료"일 때
- Then 상태를 이전 단계로 되돌릴 수 없다 (단방향 진행)
- Or 관리자가 명시적으로 되돌리기를 선택할 수 있다 (구현 결정에 따라)

## Epic A3: 테이블 관리

> 관리자가 테이블 초기 설정, 주문 삭제, 세션 관리를 수행할 수 있다.

### Story A3-1: 테이블 초기 설정
**As a** 매장 관리자로서,
**I want** 테이블 태블릿을 초기 설정하기를,
**So that** 고객이 해당 테이블에서 자동 로그인으로 주문할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 설정 시나리오*
- Given 관리자가 테이블 관리 화면에 있을 때
- When 테이블 번호와 비밀번호를 입력하고 설정 버튼을 클릭하면
- Then 16시간 유효한 테이블 세션이 생성된다
- And 설정 정보가 해당 태블릿의 localStorage에 저장된다
- And 이후 태블릿 접속 시 자동 로그인이 활성화된다

*중복 설정 시나리오*
- Given 이미 활성 세션이 있는 테이블에 재설정을 시도할 때
- Then 확인 팝업이 표시된다 ("기존 세션을 종료하고 새로 설정하시겠습니까?")
- When 확인하면 기존 세션이 종료되고 새 세션이 생성된다

### Story A3-2: 주문 삭제 (직권 수정)
**As a** 매장 관리자로서,
**I want** 잘못된 주문을 삭제하기를,
**So that** 주문 오류를 수정하고 정확한 금액을 유지할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 삭제 시나리오*
- Given 관리자가 테이블 주문 상세 화면에 있을 때
- When 특정 주문의 삭제 버튼을 클릭하면
- Then "이 주문을 삭제하시겠습니까?" 확인 팝업이 표시된다
- When 확인을 클릭하면
- Then 주문이 즉시 삭제된다
- And 테이블 총 주문액이 재계산된다
- And 성공 피드백 메시지가 표시된다

*삭제 실패 시나리오*
- Given 주문 삭제 중 서버 오류가 발생하면
- Then "주문 삭제에 실패했습니다." 에러 메시지가 표시된다
- And 주문은 그대로 유지된다

### Story A3-3: 테이블 이용 완료 처리
**As a** 매장 관리자로서,
**I want** 손님이 나간 테이블을 이용 완료 처리하기를,
**So that** 새 손님이 이전 주문 내역 없이 깨끗한 상태에서 시작할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 완료 처리 시나리오*
- Given 관리자가 테이블 카드에서 "이용 완료" 버튼을 클릭했을 때
- Then "테이블 이용을 완료 처리하시겠습니까?" 확인 팝업이 표시된다
- When 확인을 클릭하면
- Then 현재 세션의 모든 주문이 과거 이력으로 이동된다
- And 테이블 현재 주문 목록이 초기화된다
- And 총 주문액이 0원으로 리셋된다
- And 성공 피드백 메시지가 표시된다

*주문 없는 테이블 완료 처리 시나리오*
- Given 주문이 없는 테이블에 이용 완료를 시도할 때
- Then 확인 팝업 없이 즉시 처리되거나 "주문 내역이 없습니다" 안내가 표시된다

### Story A3-4: 과거 주문 내역 조회
**As a** 매장 관리자로서,
**I want** 테이블의 과거 주문 이력을 조회하기를,
**So that** 매출 현황을 파악하고 분쟁 시 근거 자료로 활용할 수 있다.

**Priority**: Should Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 조회 시나리오*
- Given 관리자가 테이블 카드에서 "과거 내역" 버튼을 클릭했을 때
- Then 해당 테이블의 과거 세션 주문 목록이 시간 역순으로 표시된다
- And 각 주문에는 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다

*날짜 필터 시나리오*
- Given 과거 내역 화면에서 날짜 범위를 선택했을 때
- Then 해당 기간의 주문만 필터링되어 표시된다

*내역 없는 시나리오*
- Given 해당 테이블에 과거 이력이 없을 때
- Then "과거 주문 내역이 없습니다" 메시지가 표시된다

## Epic A4: 메뉴 관리

> 관리자가 매장 메뉴를 동적으로 추가, 수정, 삭제할 수 있다.

### Story A4-1: 메뉴 등록
**As a** 매장 관리자로서,
**I want** 새 메뉴를 등록하기를,
**So that** 고객이 새로운 메뉴를 주문할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*정상 등록 시나리오*
- Given 관리자가 메뉴 관리 화면에서 "메뉴 추가" 버튼을 클릭했을 때
- Then 메뉴 등록 폼이 표시된다 (메뉴명, 가격, 설명, 카테고리, 이미지 업로드)
- When 모든 필수 항목을 입력하고 저장 버튼을 클릭하면
- Then 새 메뉴가 등록되고 메뉴 목록에 표시된다
- And 고객 화면에서 즉시 조회 가능하다

*유효성 검증 시나리오*
- Given 메뉴명이 비어 있을 때
- Then "메뉴명은 필수 항목입니다" 에러 메시지가 표시된다
- Given 가격이 0 이하이거나 숫자가 아닐 때
- Then "올바른 가격을 입력해 주세요" 에러 메시지가 표시된다

*이미지 업로드 시나리오*
- Given 관리자가 이미지 파일을 선택했을 때
- Then 이미지 미리보기가 표시된다
- When 저장하면 이미지가 서버에 업로드되고 메뉴에 연결된다
- And 허용 형식(jpg, png, webp)이 아닌 파일 선택 시 에러 메시지가 표시된다
- And 파일 크기 초과 시 에러 메시지가 표시된다

### Story A4-2: 메뉴 수정 및 삭제
**As a** 매장 관리자로서,
**I want** 기존 메뉴를 수정하거나 삭제하기를,
**So that** 메뉴 정보를 최신 상태로 유지할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*수정 시나리오*
- Given 관리자가 메뉴 목록에서 수정 버튼을 클릭했을 때
- Then 기존 정보가 채워진 수정 폼이 표시된다
- When 내용을 변경하고 저장하면
- Then 변경 사항이 즉시 반영된다
- And 고객 화면에서도 업데이트된 정보가 표시된다

*삭제 시나리오*
- Given 관리자가 메뉴 삭제 버튼을 클릭했을 때
- Then "이 메뉴를 삭제하시겠습니까?" 확인 팝업이 표시된다
- When 확인하면 메뉴가 삭제된다
- And 고객 화면에서 해당 메뉴가 더 이상 표시되지 않는다

*주문된 메뉴 삭제 시나리오*
- Given 현재 활성 주문에 포함된 메뉴를 삭제하려 할 때
- Then 경고 메시지가 표시된다 ("이 메뉴는 현재 주문에 포함되어 있습니다")
- And 관리자가 강제 삭제를 선택할 수 있다

### Story A4-3: 메뉴 노출 순서 조정
**As a** 매장 관리자로서,
**I want** 메뉴 노출 순서를 조정하기를,
**So that** 인기 메뉴나 추천 메뉴를 상단에 배치할 수 있다.

**Priority**: Could Have
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:

*순서 조정 시나리오*
- Given 관리자가 메뉴 관리 화면에 있을 때
- When 메뉴 항목을 드래그 앤 드롭하거나 순서 버튼(위/아래)을 클릭하면
- Then 메뉴 순서가 변경된다
- And 고객 화면에서도 변경된 순서로 표시된다
