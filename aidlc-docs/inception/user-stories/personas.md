# Personas - 테이블오더 서비스

---

## Persona 1: 고객 (Customer)

**이름**: 김민준
**나이**: 28세
**직업**: 직장인

### 특성
- 스마트폰과 태블릿 사용에 익숙한 디지털 네이티브
- 빠른 서비스를 선호하며 대기 시간에 민감
- 직관적인 UI를 기대하며 복잡한 조작을 싫어함
- 주문 후 음식이 언제 나오는지 궁금해함

### 목표
- 직원을 부르지 않고 즉시 원하는 메뉴를 주문하고 싶다
- 메뉴 사진과 설명을 보고 신중하게 선택하고 싶다
- 주문한 내역과 금액을 언제든지 확인하고 싶다
- 주문이 잘 접수되었는지 확인하고 싶다

### 불편사항 (Pain Points)
- 바쁜 시간대에 직원을 부르기 어려움
- 메뉴판이 없거나 글씨가 작아 읽기 불편함
- 주문이 제대로 전달되었는지 불안함
- 추가 주문 시 다시 직원을 불러야 하는 번거로움

### 사용 시나리오
테이블에 앉으면 태블릿 화면에 메뉴가 바로 표시됨. 카테고리를 탐색하며 원하는 메뉴를 장바구니에 담고, 최종 확인 후 주문. 주문 후 내역 탭에서 상태를 확인.

## Persona 2: 매장 관리자 (Store Admin)

**이름**: 박서연
**나이**: 35세
**직업**: 카페/레스토랑 운영자

### 특성
- 매장 운영 경험 5년 이상
- 피크 타임에 여러 테이블을 동시에 관리해야 함
- 직원 수를 최소화하여 인건비를 절감하고 싶어함
- 기술에 익숙하지만 복잡한 시스템보다 직관적인 도구를 선호
- 주문 누락이나 실수에 민감

### 목표
- 모든 테이블의 주문 상황을 한눈에 파악하고 싶다
- 신규 주문이 들어오면 즉시 알아채고 싶다
- 메뉴를 쉽게 추가/수정/삭제하고 싶다
- 테이블 정산 및 이용 완료 처리를 빠르게 하고 싶다
- 과거 주문 내역을 조회하여 매출을 파악하고 싶다

### 불편사항 (Pain Points)
- 여러 테이블에서 동시에 주문이 들어올 때 놓치는 경우 발생
- 직원이 주문을 잘못 전달하는 실수
- 메뉴 가격 변경 시 종이 메뉴판을 모두 교체해야 하는 번거로움
- 테이블 정산 후 이전 주문 내역이 섞여 혼란스러움

### 사용 시나리오
매장 오픈 전 관리자 화면에 로그인. 각 테이블 태블릿 초기 설정. 영업 중에는 대시보드에서 실시간으로 주문을 확인하고 상태를 업데이트. 손님이 나가면 테이블 이용 완료 처리. 필요 시 메뉴 관리 화면에서 메뉴 수정.
